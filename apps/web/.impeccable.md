## Design Context

### Users

Same audience as `web3insight-dashboard`: ecosystem foundations, grant reviewers, DAO ops, web3 talent scouts, bounty hunters, and individual developers using their profile as a portfolio. Usage spans focused daytime review sessions and late-night hackathon / research use. Sessions are data-intensive and often comparative.

This package is the **public-facing surface** — landing, profile analysis, share-outs. It's where someone *arrives* at Web3Insight (via a shared profile link, a search, a tweet) before going deeper into the dashboard. It must function as both a marketing entry point and a working tool for quick profile lookups.

Core job-to-be-done: *learning more about a web3 subject* — a dev, a repo, an ecosystem — with enough signal density to form a defensible judgment, while also being comfortable to link-share and land on cold.

### Brand Personality

Three words: **rigorous, on-chain-native, irreverent.** (Identical to dashboard — one product, one voice.)

- **Rigorous** — numbers and methodology are visible, not hidden behind hero charts. Users can trace a metric back to its source.
- **On-chain-native** — addresses, hashes, repo SHAs are first-class citizens, styled as such. The interface speaks the audience's vocabulary without explaining it.
- **Irreverent** — dry wit in empty states, editorial cadence in annotations, opinionated about what matters. Playful through typographic and editorial choices, not emoji, mascots, or color confetti.

Emotional goal: the user feels *equipped* — like they have the good seat at the table.

### Aesthetic Direction

**Engineering-blueprint / technical-drawing, in the spirit of Cloudflare's Sandbox SDK page.** This surface keeps the dashboard's rigor but reframes it through a schematic / architectural-drawing visual language: hairline black rules on warm cream, modular outlined panels laid out like a PCB or a 1970s IBM system manual, handwritten route-labels ("start" / "finish") connecting panels with drawn traces, and one rare accent color used like a copper trace on a circuit board.

Primary reference: **https://sandbox.cloudflare.com/** — concretely observed:
- Warm cream-white base (not pure white), black hairline borders everywhere as the primary structural element
- Page laid out as a modular grid of outlined rectangular panels, each with its own micro-composition
- Massive outlined display type with stacked / woven repetition (the word "sandbox" rendered as many layered outline instances creating a textured silhouette)
- Dotted grids and diagonal-hatch fills used as panel-background texture (not decoration on cards)
- Orange (`#F38020`-family) appears ONLY as: circuit-trace routing lines with node dots, string-literal highlights in code, and a small logo mark — never as a surface wash
- Monospace (pixel-terminal character, Departure Mono-adjacent) used for code blocks, CLI examples, and small technical labels
- Hand-drawn marker labels in a slanted script ("start", "finish") adjacent to drawn paths — the only "warm" element, and it earns its place
- Tick marks along page edges, like an architectural drawing's ruler border
- Code block syntax: bold black keywords, orange strings, muted gray comments — high contrast, no color confetti

Secondary references: Observable notebooks (annotation-first data presentation), the Financial Times' visual-journalism pieces (editorial confidence with dense data), vintage Braun / Dieter Rams technical manuals, PCB silkscreen layouts.

**Anti-references** (do not drift toward these):
- Generic Web3 landing slop (purple-to-blue gradients, neon accents on dark, glassmorphic cards with glow borders)
- Generic SaaS analytics templates (Mixpanel / Amplitude / "KPI-card-grid-with-sparkline")
- Bloomberg Terminal cosplay (amber-on-black, all-caps headers everywhere)
- Vercel/Linear/Geist monoculture — even though we ship Geist today, treat it as legacy to migrate off
- Anything that would look indistinguishable from another AI-generated dashboard

**Theme:** Both light and dark are first-class. **Light is default** — the blueprint aesthetic *lives* in light mode (cream ground, black hairlines, teal traces). Dark mode is a dedicated reinterpretation, not an inverted light theme: near-black ground, off-white hairlines, teal traces that now read as glowing instead of inked. Both must be hand-tuned. Dark is offered for late-night research; light is where marketing, profile shares, and grant-memo contexts land.

**Typography direction** (avoid reflex defaults — Geist, Inter, Plex, Syne, Fraunces, Playfair, DM Sans, etc.). The Cloudflare Sandbox page uses essentially *two* voices: heavy outlined display + pixel-terminal mono. We match that two-voice structure, with one added body face for longer prose:
- *Display (outlined / stacked hero)* — a heavy grotesque that can survive being rendered as a layered outline (stroked, repeated, woven). The display moments on this site should feel like architectural lettering, not marketing hero type. Candidate: **Bricolage Grotesque** (variable, optical sizing) or a heavier grotesque such as **NaN Tresor**-style / **ABC Monument Grotesk**-alternative from open catalogs.
- *Body / UI* — a neutral humanist sans for running prose and controls, warm enough to sit on cream without looking clinical. Candidate: **Basier Circle / GT Walsheim**-style, or **ABC Diatype**-alternative.
- *Mono (primary technical voice — high-usage, not an accent)* — a pixel-terminal or square-grotesque mono used heavily for addresses, hashes, commands, code blocks, CLI prompts, data-point labels, and any technical chrome. This face does a lot of work and must be chosen with care. Candidate: **Departure Mono** (pixel-terminal, fits the blueprint vibe), **Commit Mono** (calmer), or **IBM 3270**-style for a stronger vintage-terminal moment.
- *Handwritten micro-label (single-use accent)* — a slanted marker / technical-sketch script for the rare annotation moment where a drawn trace ends at a label (e.g. `start`, `finish`, `you are here`). This is a spice, not a food group: one or two instances per page. Candidate: **Caveat**-alternative or a genuine technical-marker face; must look drawn, not cursive.

The current Geist pairing should be treated as legacy and phased out at the next layout iteration. Mono is a primary voice here (unlike the dashboard, where it's a specimen) — but still never used as lazy "this is technical" decoration on non-technical content.

**Color direction:**

The Cloudflare Sandbox page is essentially a *three-ink* system: black (hairlines, type, panel borders), cream (ground), and orange (traces + string literals + the single logo mark). We adopt that three-ink discipline, substituting teal for orange — because teal is the Web3Insight logo hue and must remain the brand anchor across surfaces.

- **Ground — warm cream, not white.** Light mode base `~oklch(97% 0.004 180)` (almost white, faintly cool-tinted toward teal hue 180). Avoid the pure `oklch(0.98 0 0)` currently in `globals.css`.
- **Ink — near-black, slightly cool-tinted.** `~oklch(18% 0.008 180)` for borders, body type, hairlines. No pure black.
- **Trace — teal, rare and precise.** `#0F766E` ≈ `oklch(47% 0.08 180)`. This is the only chromatic hue on the page, and it appears ONLY as:
  - Routing / connector lines between panels (drawn as SVG with small node dots, like PCB traces)
  - String literals and URL highlights in code blocks
  - The active link underline, the one primary CTA, the one highlighted chart line
  - The logo mark
  Do not fill panel backgrounds with teal. Do not use teal as a surface wash. Do not use teal gradients.
- Build a full teal ramp (50 → 950) in OKLCH with reduced chroma at the extremes, but 90% of usage should be the mid-ramp `~500` value.
- **Tint all neutrals toward teal hue** (hue ~180, chroma 0.004–0.012). Light surfaces cool slightly toward teal; dark surfaces cool toward teal.
- Dark mode trade: ground becomes `~oklch(14% 0.008 180)`, hairlines become `~oklch(75% 0.005 180)`, teal stays at mid-ramp and now reads as emissive rather than inked.
- No secondary chromatic accent on this surface. The dashboard has amber-ochre for warnings; `.ai` is a cleaner two-ink + teal system. If a warning is needed, use ink weight or hatch-pattern emphasis, not a second hue.
- Retire the current green accent `oklch(0.7 0.15 145)` in `globals.css` — it's not the brand and conflicts with the palette above.
- **Banned color moves:** cyan-on-dark, purple-to-blue gradients, neon green "terminal" text, teal-to-indigo gradients, any gradient fill on text, any CSS `background-clip: text` with a gradient, colored side-stripe borders (`border-left:` > 1px) as accents. See Impeccable absolute bans.

### Design Principles

1. **The page is a blueprint, not a brochure.** Layout is a modular grid of outlined rectangular panels with hairline borders. Each panel carries one idea: a hero number, a profile card, a code sample, a chart, a labeled diagram. No rounded-corner "cards" with soft shadows — use precise 1px hairline borders. Radius is 0px or 2px, not 8px+.

2. **Traces connect the ideas.** Where two panels are related (e.g. "enter handle" → "get report" → "share"), draw a literal connecting line — teal SVG trace with small filled node dots at endpoints and bends, PCB-style. This is how the page tells its story. Do not use arrows with arrowheads; use traces with endpoint dots.

3. **Display type is stacked / outlined, not filled.** Hero moments render the word as many repeated outline instances, woven vertically (the "sandbox" treatment). Body and label type remain solid. This gives one unforgettable per-page moment without a gradient or glow.

4. **Mono is the technical voice, sans is the editorial voice.** Addresses, hashes, repo SHAs, commands, filenames, data columns, and small technical labels are mono. Running prose, section headings, and CTA labels are sans. They share the page without fighting.

5. **Texture comes from pattern fills, not color.** Dotted grids and diagonal-hatch fills are the only "decoration." Use them inside panels as ground texture (not on every panel — reserve for panels that represent background or negative-space modules in the layout grid).

6. **One rare teal.** Teal appears as traces, as string literals in code, as the single active link underline, as the one chart line the user is meant to follow, as the primary CTA fill. Never as a surface wash, never as a gradient, never on body text.

7. **Edge chrome tells you this is a technical drawing.** Thin tick marks along the outermost page edges (like an architectural drawing's rule-of-thirds border). A small grid-of-squares logo mark. Small mono labels for page zones. Commit to the conceit.

8. **Handwritten labels are spice.** One or two `start` / `finish` / `you are here` marker-labels per page, placed at trace endpoints. More than two, and the page stops feeling like a schematic and starts feeling twee.

9. **Terminal-within-page.** Embed a genuine-looking mono code panel (light background, syntax-highlighted — black keywords, teal strings, muted gray comments) for command examples, API snippets, on-chain data previews. This is where the `.ai` surface earns its credibility with a technical audience.

10. **Motion is reportorial, not performative.** Staggered entrance on page load, exponential ease-out (`[0.16, 1, 0.3, 1]`), `transform` / `opacity` only. Trace lines may *draw themselves* on load (`stroke-dashoffset` animation — the existing `animate-draw` keyframe in `globals.css` is perfect for this and should be kept and extended). No bounce, no glow pulses, no shimmer-loaders on non-async content. Respect `prefers-reduced-motion`. Audit and remove `animate-shimmer` and `animate-pulse-wave` where decorative.

11. **Show the methodology.** Every metric reveals its definition on hover or adjacent annotation. Small mono labels like `source: opendigger · 30d window`. Credibility comes from visible sourcing.

12. **Empty states teach with dry wit.** *"No repos under this filter. Try widening the window, or check `ecosystem=starknet` — that one's always busy."*

### Implementation Notes (project-specific)

- `src/app/globals.css` currently defines a pure-grayscale palette with a green accent and a `--radius: 0.5rem`. Migrate: (1) palette → teal-tinted OKLCH per color direction above, (2) `--radius` → `0.125rem` (2px) max, not `0.5rem`, (3) retire green accent.
- `next-themes` is already installed — wire it for light-default with dark as a hand-tuned reinterpretation (not a simple invert). The existing `.dark` selector pattern should stay.
- Fonts are currently Geist sans + mono via the `geist` package. Plan to swap to the typography direction above — in order of priority: (1) replace mono first since mono is the heaviest-used voice here, (2) replace display/hero next for the outlined-stacked moment, (3) body sans is lowest-priority.
- `dotted-pattern` and `diagonal-lines` utilities in `globals.css` are *exactly right* for this direction — keep them and use them as panel ground fills (not per-card decoration).
- The `animate-draw` keyframe (stroke-dashoffset animation) is perfect for the trace-drawing-itself effect on load. Keep it, extend it to accept durations/delays for staggered trace animations.
- `framer-motion` is installed; standardize on exponential ease-out (`[0.16, 1, 0.3, 1]` or `ease-out-quart`) for all entrance animations. No spring physics, no bounce.
- Recharts is the charting stack here. Define a single shared teal-based color scale in a tokens file; chart axes / gridlines should be ink hairlines, series lines should be ink except for the one highlighted series which is teal.
- Plan a small reusable `<Trace>` SVG component: takes two element refs (or panel ids) and draws an orthogonal teal route with node dots between them, animated on mount. This is the primary structural decoration — worth building well once.
